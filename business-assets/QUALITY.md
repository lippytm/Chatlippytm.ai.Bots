# Chatlippytm.ai.Bots Quality Checklist

## Purpose

Keep the bot workflow system clear, practical, safe, and useful for business owners.

## Clarity checklist

- [ ] The repo explains the bot workflow purpose in plain English.
- [ ] The primary CTA is clear: Build a Bot Workflow.
- [ ] The workflow map is simple.
- [ ] The offer avoids technical overload.
- [ ] The Canva card can be understood in 3 seconds.

## Bot workflow checklist

- [ ] The bot has one clear job.
- [ ] The lead capture step is defined.
- [ ] FAQ questions are listed.
- [ ] Qualification questions are clear.
- [ ] Offer routing is defined.
- [ ] Human handoff is included.
- [ ] CRM tag is assigned.

## Compliance checklist

- [ ] No guaranteed sales claims.
- [ ] No guaranteed lead claims.
- [ ] No claim that bots replace all human service.
- [ ] Human handoff is included.
- [ ] Affiliate/tool links are disclosed where used.

## Canva quality checklist

- [ ] Headline readable on phone.
- [ ] CTA visible: Build a Bot Workflow.
- [ ] Workflow nodes are not overcrowded.
- [ ] Conversation flow is obvious.
- [ ] Exported in square, vertical, LinkedIn, and website sizes.

## GitHub quality checklist

- [ ] Bot scripts and workflows are saved in GitHub.
- [ ] Performance log is updated weekly.
- [ ] Common questions become FAQ content.
- [ ] New bot ideas become issues.

## Best practice

A useful bot starts with one clear business problem and one clear next step.
